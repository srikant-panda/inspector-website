[01] — OVERVIEW

Inspector is a modular, zero-dependency diagnostics and file management suite built for Node.js. Developed as a dev-utility package, it aggregates low-level system metrics — OS, CPU, memory, network, battery, storage — and routes them into terminal views, JSON data payloads, and styled dark-mode HTML reports. It includes an interactive file manager supporting cross-platform command scripts and folder tree-browsing.

SANDBOX BOUNDARY

Programmatic operations inside the sandbox are strictly regulated by the FileOps class. The private _resolveSafe() helper processes all incoming paths relative to the sandbox root. If a path tries to escape (e.g. ../../etc/passwd), a Sandbox escape blocked error is thrown immediately.

[04] — CODE FLOW

Three workflows.
One render function.

Every redraw request in sysinspector/src/cli.js is routed to a single, central function: renderMenu(). Event handlers only mutate state — the render function constructs the entire frame in memory and prints it atomically.

A

Main Menu Interaction Loop

source: cli.js · keypressLoop()
1
APP STARTUP

Entry via index.js main() — arguments parsed, mode dispatched.

2
RENDER MENU

renderMenu() constructs the full frame string (header, layout pane, details box) in memory and prints atomically.

3
WAIT KEY

Keypress listener captures input. Guard verifies key is present before accessing properties.

4
ARROW MOVE / ENTER SELECT

Arrow keys mutate the selected menu index. Enter triggers executeAction().

5
UPDATE / EXECUTE

State change requested from renderMenu() — loop continues. Ctrl+C restores terminal mode and exits 0.

B

System Diagnostics Pipeline

source: sysinfo.js · gatherSystemInfo()
1
SELECT OS INFO

User selects a diagnostic category from the menu (OS, CPU, Memory, Disk, Battery, Network).

2
QUERY SYSINFO.JS

Calls gatherSystemInfo() wrapped in safe() — failures fall back to 'N/A' rather than crashing.

3
FORMAT OUTPUT

Formats ANSI layout and ASCII OS logo via cli.js rendering layer.

4
WRITE EXPORTS

Writes <name>/<name>.json and HTML report via renderInfoHtml() into the sandbox subdirectory.

5
DISPLAY SCROLL

Opens the Scrollable Output viewer with arrow-key navigation.

C

File Manager Command Flow

source: filemanager/index.js · startFileManager()
1
ENTER FILEMANAGER

startFileManager() invoked from main menu — shell prompt initialised.

2
READ PROMPT

Captures user input line, preserving quoted spaces.

3
PARSE COMMAND

parseCommand() splits tokens while preserving quoted arguments.

4
NORMALIZE ARGS

Windows aliases mapped — dir → ls, /s → -rf, md → mkdir, del → rm, ren → mv.

5
DISPATCH

Routes to read/exit or destructive (rm) branch. Destructive ops require -r/-rf for folders.

6
EXECUTE & LOG

Runs against the host system relative to current directory. Pushes audit action to fileOps.logAction().

SINGLE-RENDER-FUNCTION PATTERN

State changes and terminal writes are separated.

Early iterations had console output scattered throughout choice-handling handlers, causing menu flickering, overlapping text lines, and layout breakage when resizing terminal windows. Event handlers now only mutate state (like the selected menu index), then request an update from renderMenu(), which prints the entire frame atomically using process.stdout.write('\x1b[H\x1b[J' + frame).

[06] — ERROR & SAFETY HANDLING

Defensive by default.
Crash-proof by design.

Querying low-level OS structures can throw unexpected errors on unsupported platforms or under restricted permissions. Inspector wraps every gather function in safe() and gates every destructive file operation behind a confirm flag.

FAILURE SCENARIO LOG MATRIX
Scenario Code Behavior Handler
OS CPU/Memory Query Fails Returns 'N/A' gracefully sysinfo.js → safe()
Sandbox Path Escape Throws Sandbox escape blocked fileOps.js → _resolveSafe()
Update without Confirm Throws Update blocked exception fileOps.js → update()
Delete without Confirm Throws Delete blocked exception fileOps.js → delete()
Command Timeout Throws dynamic timeout exception sysinfo.js → execWithTimeout()
Visual Feature Load Error Displays error panel in Scroll View cli.js → executeAction().catch()
Ctrl+C Force Quit Restores Terminal mode, exits with 0 cli.js → keypressLoop() onKeypress
Fatal Process Error Logs error to stderr, exits with 1 index.js → main().catch()
Invalid Directory Listing Throws Not a directory / Directory not found fileOps.js → list()
SAFETY PHILOSOPHY
01

Gracious Fallbacks via safe()

The utility helper safe(fn, fallback = 'N/A') in sysinfo.js wraps every metrics gathering function. If it throws, the CLI displays N/A rather than terminating the process.

02

Path Traversal Protection

CRUD operations evaluate path destinations via _resolveSafe(). Paths are resolved with path.resolve() and checked against the sandbox root. Implemented on create(), read(), update(), delete(), and list().

03

Programmatic Confirm Gating

Accidental file overrides are blocked inside FileOps's update and delete operations. Functions throw a Confirm flag not set error unless the parameter confirm === true is passed.

04

Keypress Crash Guards

In Node.js CLI keypress listeners, users pressing hotkeys or mouse buttons emit key objects with undefined fields. Guards verify that key is present before accessing properties — ignoring raw mouse escape codes.

05

Ctrl+C Graceful Exit

Force-quit restores terminal mode and exits with code 0 — preventing raw-mode terminal corruption on the host shell session.
